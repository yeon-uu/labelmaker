## 타이포
- UI: Pretendard, system-ui. 본문 16px. 스케일 12/14/16/18/24/32 밖 금지.
- 라벨 폰트(출력물 전용): 'Jua', 'Gaegu' (Google Fonts) + 기본 고딕.
- `word-break: keep-all`, 행간 1.5~1.6.

## 간격·형태
- 간격 4/8 배수만: 4, 8, 12, 16, 24, 32, 48.
- radius: 카드·키 8px / 기기·라벨 16px / 버튼 높이 44px(터치 대상).
- 깊이: 1px 저대비 보더 + 표면 명도 단계로 통일. 카드 그림자 금지 (기기 PNG에 구워진 그림자만 예외).

## 모션
- UI 전환 150ms / 강조 300ms. hover에 레이아웃 밀림 금지.
- 예외 토큰: `--anim-print: 1400ms ease-out` — 라벨 출력 "주르륵"은 이 사이트의 시그니처 연출로 단 하나의 긴 애니메이션. 그 외 400ms 초과 금지.
- 키 눌림: transform scale(0.96) + 명도 변화, 80ms.

## 대담함은 한 곳
출력 애니메이션(주르륵 + 완성 토스트)에만 화려함 허용. 나머지 UI는 조용하게. 그라데이션 남발·이모지 아이콘·카드 그림자 금지.

## 2026-07-04 (2차) — UI 전면 재구성 추가 토큰

### 기기 내장형 전환
컨트롤 패널(글리프/폰트/테이프/프레임/기호 카드) 전부 삭제. 모든 조작을 `.machine-body` 내부 absolute 배치로 이동(실물 라벨기 UX). 화면에는 기기 하나만 남는다.

### 키보드 컴팩트 치수
- 키 시각 크기: 36~40px (기존 44px 고정 대신, 좁은 키보드 패널에 3줄+하단줄 배치 위해 축소).
- 히트 영역은 44px 유지 — 시각 크기와 별개로 패딩/투명 확장 영역(`::before` 또는 padding)으로 44px 이상 확보.
- 키보드 grid는 패널(left 0.83% / top 32.30% / width 93.60% / height 36.94%, 몸체 기준) 안에 꽉 채운다.

### 기능키 줄 (MODE/방향키)
- 5개 소형 버튼(MODE, ▲▼◀▶), 키보드 패널 최상단 줄에 배치(LCD 아래 틈 공간 부족 판단).
- 크기: 시각 32px, 히트 44px(패딩 확장).

### 설정 모드 (state.mode)
- `state.mode`: `'text' | 'settings'`. `state.settingIndex`: 0~3 (폰트 → 테이프색 → 프레임 → 사이즈 순환).
- MODE 버튼으로 text ↔ settings 전환. ▲▼로 항목 이동(순환), ◀▶로 값 순환(순환).
- LCD 표시 형식(설정모드): `"{항목명}: {현재값} ◀▶"` 1줄.

### 라벨 사이즈 2종 (신규)
- **긴 스트립**(기존): width 280 / height 100 (표시 px, scale 전).
- **정사각**: width 160 / height 160. 여러 줄 텍스트 중앙 배치, `render.js`의 줄바꿈 로직이 maxWidth=160-padding 기준으로 동일하게 동작(정사각 폭이 좁아 자연히 더 많이 줄바꿈됨).
- 다운로드 PNG는 표시 px의 3배 해상도 유지.

### 라벨 모서리 radius 축소 (신규)
- 기존 `--radius-device: 16px`를 라벨 전용으로 쓰지 않고, 라벨 자체 radius는 `--radius-label: 3px`로 신설(거의 직각, 스트레이트 컷 느낌). 기기 본체 radius 개념(`--radius-device`)은 유지하되 라벨에는 미적용.
- `render.js drawLabel()`의 기본 `radius` 인자를 16 → 3으로 변경.

### 라벨 텍스처 (신규)
- SVG `feTurbulence` grain 오버레이, 불투명도 4%(3~6% 범위 중간값) — `--label-grain-opacity: 0.04`.
- 화면: `.label-canvas`에 CSS `filter` 또는 오버레이 `::after` + SVG filter로 근사, 혹은 canvas 자체에 노이즈를 합성(택1, 구현 시 확정해 아래에 갱신).
- Canvas PNG: 오프스크린 캔버스에 랜덤 그레이스케일 노이즈 픽셀을 만들어 `globalAlpha: 0.04`로 라벨 영역에만 합성 후 클리핑.

### 잉크 번짐 (신규)
- 텍스트에 `shadow-blur: 0.3px` 수준의 미세 blur만 추가(Canvas `ctx.shadowBlur`), 색은 잉크색과 동일(번짐이지 그림자 아님). 화면 미리보기는 LCD 텍스트에는 적용하지 않음(LCD는 디지털 표시라 인쇄 질감과 무관) — 오직 출력된 라벨(Canvas)에만.

### 완성 라벨 그림자
- `box-shadow: 0 2px 6px rgba(58,51,48,0.08)` 수준의 저채도 저투명도 1개만. 광택/그라데이션 금지(기존 규칙 재확인).

### 사운드 파라미터 (js/sound.js, 문서화 목적 — 실제 값은 코드가 원본)
- 키 입력음: sine/triangle 600~900Hz 랜덤(±10~15%), attack 5ms / release 40ms, 총 40~80ms.
- 스페이스/지우기: 400Hz대, 동일 엔벨로프.
- 출력 시작음: 100→200Hz linearRamp, 150ms.
- 인쇄 사운드: 화이트노이즈 버스트 짧게 게이팅, 6~8회 반복(감김 애니메이션과 동기화).
- 볼륨 기본 낮게, 음소거 상태 `localStorage['labelmaker:muted']`에 저장.
- AudioContext는 첫 사용자 상호작용 시 생성 + resume().

### 출력 모션 리듬 (신규)
- `--anim-print: 1400ms` 유지, 단일 ease 대신 6~8단계 `@keyframes`로 "감기며 올라옴"(멈칫 반복) + 마지막 2~3px 오버슈트.
- 슬롯 위치에 `overflow:hidden` 클리핑 컨테이너 신설(라벨이 슬롯 아래에서 보이지 않고 슬롯 틈에서만 나오는 것처럼).
- perspective + rotateX 2~4deg → 0deg 수렴(모서리 말림 느낌), 슬롯 위 옅은 접촉 그림자 1개.
- machine-wobble과 인쇄 사운드를 같은 6~8스텝 타이밍에 동기화.
