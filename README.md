# 라벨메이커 (Label Maker) 🏷️

> 화면 속 귀여운 라벨프린터로 문구·이모지를 찍어 **라벨 스티커를 만들고 PNG로 저장**하는 인터랙티브 웹.
> 실물 감열 라벨기처럼 키를 누르고, LCD에 픽셀 글씨가 뜨고, 영수증처럼 라벨이 두두두 올라옵니다.

## 이렇게 동작해요

```
문구 입력 (온스크린 두벌식 / 실키보드 IME)
→ 환경 버튼으로 테이프색·프레임·사이즈 선택
→ 이모티콘 키로 이모지 자판 전환·삽입
→ 핑크 버튼 → "출력하시겠습니까? 예/아니오"
→ 슬롯에서 라벨이 영수증처럼 올라옴 → PNG 저장
```

## 주요 기능

| 기능 | 설명 |
|------|------|
| ⌨️ **이미지 키보드** | 실사 감열 라벨기 렌더 위에 투명 히트존을 얹어 진짜 키를 누르는 느낌. 한글·영문·이모지 3개 레이어를 이미지 교체로 전환 |
| 🔤 **두벌식 한글 조합** | 초성·중성·종성 오토마타 직접 구현, 온스크린 클릭과 실키보드 IME 모두 지원 |
| 📟 **픽셀 LCD** | 계산기식 단일 라인 가로 스크롤 — 넘친 글자는 창 밖으로 밀려 숨고 방향키로 되돌아봄. 픽셀 폰트(Galmuri) |
| 🎨 **꾸미기** | 테이프색 6종 · 프레임 6종 · 사이즈(스트립/정사각) · 이모지 스프라이트 |
| 🧾 **영수증 출력** | 감열 프린터처럼 단계적으로 끊기며 올라오는 애니메이션 + 멀티라인 자동 줄바꿈으로 라벨이 길어짐 |
| 🔊 **ASMR 사운드** | Web Audio로 합성한 키 입력음·출력음 (음소거 토글) |
| 💾 **PNG 저장** | Canvas로 렌더한 투명 배경 라벨을 다운로드, 저장 후 초기 화면으로 리셋 |

## 기술 노트

- **정렬**: AI 생성 기기 이미지의 여백을 트림한 뒤 `<img>` + `position:absolute; inset:0` 오버레이 구조로, 좌표 %가 이미지에 정확히 대응하게 설계. 각 히트존 중심이 실제 키캡 위에 있는지 픽셀 스캔으로 자동 검산.
- **라벨 렌더**: 사진 합성 대신 Canvas로 직접 그림 — 테이프색 배경 + 비닐 광택 + 도트 폰트(잉크색 단색) + 이모지 스프라이트.
- **의존성 0**: 프레임워크·번들러 없이 브라우저 표준(Canvas, Web Audio)만 사용.

## 만든 과정 — AI 오케스트레이션

나는 백엔드와 보안을 공부한다. 프론트엔드는 내 영역이 아니다. 그래서 코드를 직접 타이핑하는 대신 Claude Code에 구현을 맡기고, 나는 컨셉과 디자인 방향을 정하고 결과물을 판정하는 쪽을 맡았다. 브라우저로 직접 열어 확인하고, 어긋난 데를 짚어 다시 시키기를 반복했다. 매끄럽지 않았다. 막힌 데가 더 많았고, 아래는 그중 넷이다.

### ① 키보드가 사진과 계속 어긋남

- **문제**: 라벨기 사진 위에 CSS로 키를 얹으니 누르는 자리와 보이는 키가 안 맞음. 키 위치를 기기 몸체 기준 %로 쟀는데, 화면에선 투명 여백까지 포함한 전체 이미지에 그 값을 적용해 여백 비율만큼 전부 밀림.
- **해결**: 이미지 여백을 잘라내고 `<img>` 위에 `inset:0` 오버레이를 덮어 좌표가 이미지에 정확히 대응하게 변경. 각 클릭 영역 중심 픽셀이 실제 키 위에 있는지 코드가 자동 검산.
- **배운 것**: 퍼센트 좌표는 어떤 컨테이너를 기준으로 도는지가 전부다. 정렬은 눈이 아니라 수치로 검증해야 안 흔들린다.

### ② 라벨을 사진에서 못 뽑음

- **문제**: 실물 라벨지 사진에서 사각형을 누끼로 따니 4도쯤 기울고 배경 회색이 묻어 지저분.
- **해결**: 사진을 버리고 라벨을 Canvas로 직접 그림. 테이프색 배경에 도트 폰트로 글자를 찍고 광택을 한 줄 얹음.
- **배운 것**: AI가 만든 사진에서 깨끗한 요소를 잘라 쓰는 건 세 번 다 실패했다. 코드로 그리자 오히려 감열 라벨 느낌에 가까워졌다.

### ③ 출력 버튼을 눌러도 라벨이 안 나옴

- **문제**: 라벨 요소는 생성되는데 화면에 없음. 클리핑 영역 바깥 아래에 놓였고 애니메이션 최종 상태가 이동 없음이라, 끝까지 재생돼도 계속 숨음.
- **해결**: 위치를 뒤집어 슬롯 위로 올라오는 최종 상태로 수정.
- **배운 것**: "요소가 있다"와 "화면에 보인다"는 다르다. DOM 존재만 확인하면 이런 버그를 놓친다.

### ④ 이모지 키를 누르면 다른 게 나옴

- **문제**: 반짝이 키 개수를 하나 잘못 세는 바람에 뒤 배열이 한 칸씩 밀려 리본 키가 통째로 사라짐.
- **해결**: 스프라이트 파일과 키보드 이미지를 나란히 열어 하나씩 대조하고 매핑을 다시 씀.
- **배운 것**: 순서에 기댄 매핑은 한 칸만 틀려도 전부 무너진다. 원본을 직접 대조하는 쪽이 빨랐다.

부딪히며 프론트의 기초도 익혔다. 퍼센트 좌표계의 기준 컨테이너, Canvas로 이미지와 텍스트 합성, CSS 클리핑과 keyframes, Web Audio 합성음, 두벌식 조합 오토마타, 정적 배포까지. AI를 부리는 요령도 하나 남았다. 요구를 중간에 조금씩 더하면 재작업이 커져서, 한 번에 완결된 사양으로 넘기는 편이 쌌다. "됐다"는 보고는 믿지 않고 매번 브라우저로 확인했다.

## 실행 방법

```bash
# 정적 파일이라 아무 정적 서버로 열면 됩니다
npx serve .
# 또는
python -m http.server 5941
```

브라우저에서 `http://localhost:5941` 접속.

## 프로젝트 구조

```
├── index.html
├── css/style.css
├── js/
│   ├── app.js       # 상태·입력·키보드 레이어·설정·출력 플로우
│   ├── hangul.js    # 두벌식 조합 오토마타
│   ├── render.js    # 라벨 Canvas 렌더 (멀티라인·프레임·이모지)
│   └── sound.js     # Web Audio 합성음
├── assets/          # 기기 이미지 · 이모지 스프라이트
└── docs/            # 계획·작업일지
```
