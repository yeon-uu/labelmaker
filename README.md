# 라벨메이커 (Label Maker) 🏷️

> 화면 속 귀여운 라벨프린터로 문구·이모지를 찍어 **라벨 스티커를 만들고 PNG로 저장**하는 인터랙티브 웹.
> 실물 감열 라벨기처럼 키를 누르고, LCD에 픽셀 글씨가 뜨고, 영수증처럼 라벨이 두두두 올라옵니다.

## 이렇게 동작해요

```
문구 입력 (온스크린 두벌식 / 실키보드 IME)
→ 환경 버튼으로 테이프색·프레임·사이즈 선택
→ 이모티콘 키로 이모지 자판 전환·삽입
→ 핑크 버튼 → "출력하시겠습니까? 예/아니오"
→ 슬롯에서 라벨이 영수증처럼 올라옴 → PNG 저장
```

## 주요 기능

| 기능 | 설명 |
|------|------|
| ⌨️ **이미지 키보드** | 실사 감열 라벨기 렌더 위에 투명 히트존을 얹어 진짜 키를 누르는 느낌. 한글·영문·이모지 3개 레이어를 이미지 교체로 전환 |
| 🔤 **두벌식 한글 조합** | 초성·중성·종성 오토마타 직접 구현, 온스크린 클릭과 실키보드 IME 모두 지원 |
| 📟 **픽셀 LCD** | 계산기식 단일 라인 가로 스크롤 — 넘친 글자는 창 밖으로 밀려 숨고 방향키로 되돌아봄. 픽셀 폰트(Galmuri) |
| 🎨 **꾸미기** | 테이프색 6종 · 프레임 6종 · 사이즈(스트립/정사각) · 이모지 스프라이트 |
| 🧾 **영수증 출력** | 감열 프린터처럼 단계적으로 끊기며 올라오는 애니메이션 + 멀티라인 자동 줄바꿈으로 라벨이 길어짐 |
| 🔊 **ASMR 사운드** | Web Audio로 합성한 키 입력음·출력음 (음소거 토글) |
| 💾 **PNG 저장** | Canvas로 렌더한 투명 배경 라벨을 다운로드, 저장 후 초기 화면으로 리셋 |

## 기술 노트

- **정렬**: AI 생성 기기 이미지의 여백을 트림한 뒤 `<img>` + `position:absolute; inset:0` 오버레이 구조로, 좌표 %가 이미지에 정확히 대응하게 설계. 각 히트존 중심이 실제 키캡 위에 있는지 픽셀 스캔으로 자동 검산.
- **라벨 렌더**: 사진 합성 대신 Canvas로 직접 그림 — 테이프색 배경 + 비닐 광택 + 도트 폰트(잉크색 단색) + 이모지 스프라이트.
- **의존성 0**: 프레임워크·번들러 없이 브라우저 표준(Canvas, Web Audio)만 사용.

## 만든 과정 — AI 오케스트레이션

나는 백엔드와 보안을 공부한다. 프론트엔드는 내 영역이 아니다. 그래서 코드를 직접 타이핑하는 대신 Claude Code에 구현을 맡기고, 나는 컨셉과 디자인 방향을 정하고 결과물을 판정하는 쪽을 맡았다. 화면을 브라우저로 열어 확인하고, 어긋난 데를 짚어 다시 시키기를 반복했다. 단계마다 커밋으로 되돌릴 지점을 남겼다.

매끄럽지 않았다. 막힌 데가 더 많았다.

**키보드가 사진과 계속 어긋났다.** 라벨기 사진 위에 CSS로 키를 그려 얹었더니 누르는 자리와 보이는 키가 안 맞았다. 좌표 기준이 서로 달랐던 게 원인이다. 키 위치는 기기 몸체 기준 퍼센트로 쟀는데, 화면에선 투명 여백까지 포함한 전체 이미지에 그 값을 적용해 여백 비율만큼 전부 밀렸다. 이미지 여백을 잘라내고 `<img>` 위에 `inset:0` 오버레이를 덮는 구조로 바꿔 좌표가 이미지에 정확히 대응하게 했다. 여기에 각 클릭 영역의 중심 픽셀이 실제 키 위에 있는지 코드가 스스로 검사하게 만들었다. 눈대중으로 넘어가던 정렬을 숫자로 잡았다.

**라벨을 사진에서 따내려다 버렸다.** 실물 라벨지 사진에서 깨끗한 사각형을 누끼로 뽑으려 했지만 결과가 4도쯤 기울고 배경 회색이 묻어 지저분했다. 사진을 포기하고 라벨을 Canvas로 직접 그렸다. 테이프색 배경에 도트 폰트로 글자를 찍고 광택을 한 줄 얹으니 오히려 감열 라벨 느낌에 가까웠다.

**출력 버튼을 눌러도 라벨이 안 나왔다.** 요소는 만들어지는데 화면에 없었다. 클리핑 영역 바깥 아래에 놓였고 애니메이션 최종 상태가 이동 없음이라, 끝까지 재생돼도 숨어 있는 구조였다. 위치를 뒤집어 슬롯 위로 올라오게 고쳤다.

**이모지 키를 누르면 다른 게 나왔다.** 반짝이 키 개수를 하나 잘못 세는 바람에 뒤 배열이 한 칸씩 밀려 리본 키가 통째로 사라져 있었다. 스프라이트 파일과 키보드 이미지를 나란히 열어 하나씩 대조하고서야 바로잡았다.

그렇게 부딪히면서 프론트의 기초를 익혔다. 퍼센트 좌표계가 어떤 컨테이너를 기준으로 도는지, Canvas로 이미지와 텍스트를 합성하는 법, CSS로 넘치는 영역을 자르고 keyframes로 동작을 쪼개는 법, Web Audio로 소리를 만드는 법, 두벌식 한글이 초성과 중성, 종성으로 조합되는 오토마타, 정적 사이트를 무료로 배포하는 법까지.

AI를 부리는 데서도 배웠다. 요구사항을 중간에 조금씩 더하면 앞 작업과 충돌해 재작업이 커진다. 원하는 걸 모아 한 번에 완결된 사양으로 주는 편이 쌌다. "됐다"는 보고는 믿지 않고 브라우저로 직접 확인했다. 실패가 반복되면 세부를 고치는 대신 접근을 통째로 바꿨다. 사진 누끼를 세 번 실패하고 코드 렌더로 갈아탄 게 그 예다.

## 실행 방법

```bash
# 정적 파일이라 아무 정적 서버로 열면 됩니다
npx serve .
# 또는
python -m http.server 5941
```

브라우저에서 `http://localhost:5941` 접속.

## 프로젝트 구조

```
├── index.html
├── css/style.css
├── js/
│   ├── app.js       # 상태·입력·키보드 레이어·설정·출력 플로우
│   ├── hangul.js    # 두벌식 조합 오토마타
│   ├── render.js    # 라벨 Canvas 렌더 (멀티라인·프레임·이모지)
│   └── sound.js     # Web Audio 합성음
├── assets/          # 기기 이미지 · 이모지 스프라이트
└── docs/            # 계획·작업일지
```
